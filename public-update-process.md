# Public Update Process

This is the required process for any change to this public repository.

The goal is simple: keep personal information out and keep the system versionable.

## Required Checklist (every time)

1. **Strip all personal references**  
   Remove names, private dates, specific life details, private preferences, and anything that belongs only in private memory or private skills.

2. **Check against the six principles**  
   Especially ownership, usable progress, built-in review, and deliberate versioning.

3. **Update the skill list in README.md** if a new skill is added or removed.

4. **Write a clear commit message** that explains the intent of the change.

5. **Quick human review BEFORE push**  
   The human must look at the final content and explicitly approve before any push happens.

## Why this process exists

Versioning only works if the public record stays clean.  
A single personal detail that leaks becomes permanent public history.  
Requiring human review *before* the push is the practical safeguard that keeps the system humble and reversible.

This process itself is versioned. Review it when it hits friction, after ~6 months, or when the model improves.
